# Подготовка к запуску продаж (Legal & Billing)

Данный документ описывает шаги, которые необходимо выполнить перед включением монетизации и приёмом реальных платежей от физических лиц.

## 1. Юридические документы

Необходимо подготовить и утвердить у юриста следующие документы:
- **Пользовательское соглашение (Оферта)** — условия предоставления сервиса.
- **Политика обработки персональных данных** — сбор, хранение, согласие.
- **Политика использования файлов cookie** — предупреждение об использовании cookies.
- **Правила возврата средств** (Refund Policy) — условия и сроки возврата денег.
- **Условия подписки (Recurring payments)** — соглашение на автоматические списания.

В коде установлены метки `TODO_LEGAL(...)` для мест, требующих заполнения реальными датами и ссылками.

## 2. Юридические данные (Реквизиты)

- [ ] Уточнить и внести в `apps/frontend/src/config/legal.ts` реальный адрес ИП (место нахождения).
- [ ] Уточнить адрес для направления претензий (почтовый адрес или email).
- [ ] Указать применяемую систему налогообложения (УСН, НПД и т.д.) для чеков.
- [ ] Указать банк и расчетный счет для вывода средств (в коде указан текущий).

## 3. Техническая интеграция платежей

- [ ] Зарегистрироваться в ЮKassa или другом платежном шлюзе.
- [ ] Настроить вебхуки (Webhooks) от платежной системы к бекенду (`/api/billing/webhook`).
- [ ] Реализовать создание платежа на стороне бекенда (YooKassa API) и возврат ссылки на оплату.
- [ ] Обработка статусов `succeeded`, `canceled`, `waiting_for_capture`.
- [ ] Настройка выдачи чеков (через онлайн-кассу, интегрированную с эквайрингом).

## 4. Скрипт проверки

Для проверки готовности кодовой базы используйте скрипт:
```bash
npm run legal:check:strict
```
Скрипт проверяет наличие `TODO_LEGAL` в коде. Если метки найдены, запуск продаж недопустим.

## 5. Включение оплат

После выполнения всех пунктов:
1. Замените все `TODO_LEGAL(...)` на реальные данные.
2. В файле `.env` установите `PAYMENTS_ENABLED=true` и `VITE_PAYMENTS_ENABLED=true`.
3. Перезапустите приложение.
