# Викторина (Buzzer MVP)

Система интерактивной викторины (брейн-ринг) в реальном времени, позволяющая ведущему создавать комнаты, а участникам — соревноваться в скорости реакции ("Кто нажмет кнопку первым").

## 🚀 Стек технологий

Проект организован как monorepo (npm workspaces) и включает в себя:
- **Frontend**: React, Vite, TypeScript, Tailwind CSS, Shadcn UI, Socket.IO Client.
- **Backend**: Node.js, Express, Socket.IO (WebSockets), TypeScript.
- **Shared**: Общие TypeScript-типы и интерфейсы для синхронизации клиента и сервера.

## 🛠 Локальный запуск и разработка

Проект можно запустить двумя способами: классическим способом через Node.js (удобно для разработки) или через Docker (удобно для быстрого тестирования в изолированной среде).

### Способ 1: С помощью Docker (Быстрый запуск)
Для этого потребуется установленный [Docker](https://www.docker.com/) и Docker Compose.

1. **Клонируйте/откройте проект** в терминале.
2. **Запустите сборку и запуск контейнеров**:
   ```bash
   docker-compose up --build
   ```
3. **Готово!** Проект будет доступен по адресу: http://localhost:8080 (через Nginx) или напрямую http://localhost:5173.

### Способ 2: С помощью Node.js (Для разработки)
Для этого потребуется установленный [Node.js](https://nodejs.org/) (рекомендуется версия 18+).

1. **Клонируйте/откройте проект** в терминале.
2. **Установите зависимости** для всего monorepo (выполняется из корня проекта):
   ```bash
   npm install
   ```
3. **Запустите режим разработки** (команда одновременно поднимет backend и frontend):
   ```bash
   npm run dev
   ```

После запуска:
- Frontend будет доступен по адресу: http://localhost:5173
- Backend (WebSocket API) будет запущен на: http://localhost:3001

## 📖 Сценарий использования (Как пользоваться)

Приложение разделено на два интерфейса: для **Ведущего** и для **Участника**.

### Роль "Ведущего" (Host)
Ведущий управляет процессом игры. Для входа в панель управления ведущий должен перейти по ссылке: **`/host`** (например, `http://localhost:5173/host`).

Для тестирования в базе данных по умолчанию (при запуске `npm run db:seed` или запуске Docker) создан тестовый аккаунт с активной подпиской:
- **Email:** `admin@example.com`
- **Пароль:** `admin123`

1. Ведущий заходит по адресу `/host`, авторизуется и нажимает "Создать комнату".
2. Система генерирует уникальный **6-значный код комнаты** (например, `XYZ123`), который ведущий сообщает участникам (выводит на экран или отправляет в чат).
3. В панели ведущего отображается список подключившихся игроков в реальном времени.
4. Ведущий задает вопрос вслух или выводит его на экран. Как только кто-то из игроков нажимает кнопку, система определяет самого быстрого.
5. Экран ведущего показывает победителя (кто нажал первым) и блокирует кнопки у остальных.
6. После того как ответ принят (правильный или нет), ведущий нажимает кнопку **"Сбросить"**, чтобы разблокировать кнопки всех участников для следующего вопроса.
7. *Дополнительно*: Ведущий может удалять (кикать) игроков из комнаты, если они подключились по ошибке или нарушают правила.

### Роль "Участника" (Participant)
Участники заходят на главную страницу приложения (корень сайта `/`, например `http://localhost:5173/`).

1. Участник открывает приложение на своем телефоне или ноутбуке.
2. Вводит код комнаты (который дал ведущий) и свое имя (никнейм).
3. Перед участником появляется большая кнопка-баззер ("ЖМИ!").
4. Как только участник знает ответ, он должен нажать на кнопку быстрее остальных. 
5. Если он был первым, его экран загорается зеленым ("Вы были первым!"). У всех остальных участников экраны блокируются ("Кнопка заблокирована", "Игрок [Имя] нажал первым").
6. Участники ждут, пока ведущий сбросит раунд, после чего кнопка снова становится активной.

## 📌 Ключевые функции (Функционал)

- **Мгновенная синхронизация**: Использование WebSockets (Socket.IO) гарантирует минимальную задержку (ping) при определении того, кто нажал кнопку первым. Разница измеряется в миллисекундах.
- **Защита от фальстартов**: Кнопки всех игроков автоматически блокируются, как только первый человек успел нажать свой баззер.
- **Адаптивный дизайн**: Интерфейс участников спроектирован так, чтобы было максимально удобно быстро нажимать кнопку с мобильного телефона (используются большие tap-зоны и защита от случайного скролла/зума).
- **Управление комнатой**: Ведущий имеет полный контроль над списком игроков и статусом раунда.

## 🌍 Развертывание на сервере (Production)

Для вывода проекта в интернет (чтобы другие люди могли подключаться с телефонов) мы используем **Docker** и **Cloudflare Tunnels**. Это позволяет получить бесплатный домен с SSL-сертификатом (HTTPS) без необходимости настраивать сложные сертификаты и открывать порты на сервере.

> **Важно:** Процесс сборки (компиляции) проекта `npm ci && npm run build` внутри Docker требует как минимум **1.5 - 2 ГБ оперативной памяти (RAM)**. Если у вашего сервера меньше памяти (например, 1 ГБ), сервер может "зависнуть" (OOM). В таком случае рекомендуется собирать образы Docker локально на компьютере и отправлять на сервер уже готовые.

### Вариант А: Быстрый туннель (Без регистрации)
По умолчанию в `docker-compose.yml` настроен быстрый туннель (Quick Tunnel). Он генерирует случайную ссылку вида `https://random-words.trycloudflare.com`.

1. На сервере выполните сборку и запуск проекта:
   ```bash
   docker-compose up --build -d
   ```
2. Посмотрите логи контейнера Cloudflare, чтобы узнать вашу публичную ссылку:
   ```bash
   docker-compose logs cloudflared | grep trycloudflare
   ```
3. Скопируйте ссылку и отправьте её участникам!

### Вариант Б: Постоянный домен (Через ваш аккаунт Cloudflare)
Если у вас есть свой домен в Cloudflare и вы хотите красивую ссылку (например, `quiz.yourdomain.com`):
1. Зайдите в панель управления Cloudflare -> Zero Trust -> Networks -> Tunnels.
2. Создайте туннель и скопируйте ваш **Tunnel Token**.
3. В файле `docker-compose.yml` измените команду для `cloudflared`:
   ```yaml
   cloudflared:
     image: cloudflare/cloudflared:latest
     command: tunnel run --token ВАШ_ТОКЕН
   ```
4. Перезапустите контейнеры: `docker-compose up -d`.
